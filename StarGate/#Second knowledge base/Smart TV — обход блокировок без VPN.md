# Smart TV — обход блокировок без VPN

> Доступ к заблокированным сайтам (Medium, Reddit, зарубежные магазины) и сайтам с геоблоком для российских IP (ti.com, electronics-related) на Smart TV без установки VPN-приложения.
> Требует работающего [[Tailscale + Mihomo на Proxmox]].

---

## Почему Zapret здесь не поможет

Zapret обходит только **DPI-замедление** — когда провайдер распознаёт трафик и режет скорость или разрывает соединение. YouTube, Discord, EFT — это его случай.

Здесь другое:

| Сайт | Тип блока | Решение |
|---|---|---|
| Medium.com, Reddit | РКН: IP-блок + DNS-блок | Прокси с незаблокированным IP |
| ti.com, mouser.com и др. | Геоблок самого сайта для RU IP | Выход через нероссийский IP |
| YouTube, Discord | DPI-замедление/разрыв | Zapret |

Оба первых случая решаются одинаково: трафик должен выйти с **нероссийского IP твоего VPS**. Mihomo уже умеет это делать — правило `MATCH,PROXY` в конфиге отправляет всё незнакомое через VLESS на VPS.

Задача — пустить трафик телика через Mihomo.

## Способ 1 — Прокси в настройках телевизора (проще всего)

Samsung (Tizen) и LG (webOS) поддерживают HTTP-прокси в сетевых настройках. Это работает для всего HTTPS-трафика без каких-либо изменений на роутере.

### Samsung (Tizen)

**Настройки → Общие → Сеть → Состояние сети → Настройки IP → Прокси-сервер**

| Поле | Значение |
|---|---|
| Прокси-сервер | IP адрес LXC с Mihomo (например `192.168.1.163`) |
| Порт | `7890` |

> Если пункта «Прокси-сервер» нет — попробовать **Настройки → Поддержка → Самодиагностика → Прокси**.

### LG (webOS)

**Настройки → Сеть → Подключение Wi-Fi (или проводное) → Расширенные настройки → Прокси**

Вручную → ввести IP и порт 7890.

### Проверка

Открыть браузер на телике → зайти на `medium.com` или `ti.com`. Если работает — готово.

> В Mihomo UI (`http://192.168.1.163:9090/ui`) в разделе **Connections** появятся соединения от IP телика.

## Способ 2 — Прозрачный прокси на роутере (если настроек нет)

Если телевизор не позволяет задать прокси вручную — настроить перенаправление трафика на роутере. Пакеты уходят с телика как обычно, роутер незаметно разворачивает их через Mihomo.

Используется `redsocks` — демон, который принимает TCP-соединения локально и пробрасывает их через SOCKS5/HTTP-прокси (Mihomo).

### 2.1 Установка redsocks на OpenWrt

```bash
opkg update
opkg install redsocks
```

### 2.2 Конфигурация

```bash
vi /etc/redsocks.conf
```

```
base {
    log_debug = off;
    log_info = on;
    log = "syslog:daemon";
    daemon = on;
    redirector = iptables;
}

redsocks {
    local_ip = 0.0.0.0;
    local_port = 12345;
    ip = 192.168.1.163;    # IP LXC с Mihomo — поменять на свой
    port = 7890;           # mixed-port Mihomo
    type = socks5;
}
```

```bash
service redsocks enable
service redsocks start
```

### 2.3 Правила nftables для телевизора

Определить IP телика:

```bash
# На роутере — посмотреть DHCP-лизинги
cat /tmp/dhcp.leases
# или в LuCI: Network → DHCP and DNS → Active DHCP Leases
```

Создать файл правил:

```bash
vi /etc/zapret/rules-tv.nft
```

```nft
table inet tv-proxy {
    chain prerouting {
        type nat hook prerouting priority dstnat;

        # Заменить на IP своего телика
        ip saddr 192.168.1.XXX tcp dport { 80, 443 } redirect to :12345
    }
}
```

> `redirect` — перенаправляет на локальный порт routers'а (redsocks), который уже форвардит в Mihomo.

Применить:

```bash
nft -f /etc/zapret/rules-tv.nft
```

Добавить в автозапуск — добавить в `/etc/init.d/zapret` в `start_service()`:

```sh
nft -f /etc/zapret/rules.nft
nft -f /etc/zapret/rules-tv.nft
```

### 2.4 Проверка

```bash
# Смотреть логи redsocks
logread | grep redsocks

# Или с телика сделать запрос и смотреть
tcpdump -i br-lan -n host 192.168.1.XXX
```

## Зафиксировать IP телика

Чтобы IP телика не менялся при каждой перезагрузке — привязать MAC-адрес к фиксированному IP через DHCP.

В LuCI: **Network → DHCP and DNS → Static Leases → Add**

| Поле | Значение |
|---|---|
| MAC-адрес | MAC телика (написан на наклейке снизу или в меню телика) |
| IP-адрес | `192.168.1.XXX` (тот что прописал в правилах) |

## Проверить что Mihomo правильно роутит

Убедиться что в конфиге Mihomo (`/etc/mihomo/config.yaml`) есть правило на случай незнакомых сайтов:

```yaml
rules:
  - IP-CIDR,192.168.0.0/16,DIRECT   # ← LAN не через прокси
  - GEOSITE,ru-blocked,PROXY         # ← РКН список через VPS
  - GEOIP,RU,DIRECT                  # ← российские IP напрямую
  - MATCH,PROXY                      # ← всё остальное (ti.com, Medium) → VPS
```

`MATCH,PROXY` — это и есть catch-all для геоблокированных сайтов. ti.com не российский IP и не в ru-blocked → попадает сюда → уходит через VPS с нероссийским IP.

## Частые проблемы

| Симптом | Причина | Решение |
|---|---|---|
| Прокси в настройках Samsung не появляется | Старая прошивка | Обновить прошивку телика |
| Сайт открывается, но видео не грузится | Видеопоток идёт мимо прокси (отдельный CDN) | Добавить CDN-домены в правила Mihomo явно |
| redsocks стартует, но соединения не проходят | Mihomo не принимает от роутера | Проверить `allow-lan: true` в config.yaml и firewall на LXC |
| ti.com открывается, но пишет «регион недоступен» | Куки/сессия сохранила RU геолокацию | Открыть в режиме инкогнито / очистить куки |
| IP телика меняется | Нет статического DHCP lease | Привязать MAC → IP через Static Leases |
